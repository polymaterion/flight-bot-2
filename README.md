# Turkmen Flights Bot

Telegram-бот для поиска билетов в Туркменистан через Travelpayouts Data API
с покупкой по партнёрской ссылке Aviasales и подписками на цену.

## Архитектура

Два независимых процесса:

- **bot.py** — сам Telegram-бот (диалог, поиск, показ карточек, создание подписок)
- **worker.py** — фоновый процесс, раз в N часов проверяет все подписки и шлёт уведомления

Они общаются через общую SQLite базу (`turkmenbot.db`). Разделение на два процесса
позволяет worker'у работать долго и независимо, не блокируя бота во время проверок.

## Логика поиска → подписка

1. Пользователь вводит откуда/куда/даты
2. Бот запрашивает `/v1/prices/cheap` у Travelpayouts
3. Если есть результаты — карточки с кнопками "Купить" (партнёрская ссылка) и "Следить за ценой"
4. Если результатов нет — предложение создать подписку с `last_price = NULL`
5. `worker.py` каждые 4 часа проходит по всем активным подпискам:
   - если `last_price` был `NULL`, а сейчас нашлись билеты → уведомление "билеты появились"
   - если `last_price` был известен, а новая цена ниже → уведомление "цена упала"
   - если цена не изменилась или выросла → тихо обновляет `last_price`, не спамит

## Настройка

1. `pip install -r requirements.txt`
2. Вписать токены в `bot.py`, `worker.py`, `travelpayouts_api.py`:
   - `BOT_TOKEN` — токен от @BotFather
   - `TOKEN` — API-токен Travelpayouts (личный кабинет → API)
   - `MARKER` — ваш партнёрский marker для ссылок

## Запуск

```bash
python bot.py       # в одном терминале / процессе
python worker.py     # в другом
```

В проде — как два systemd-сервиса или два процесса в docker-compose.

## Что нужно доработать перед продакшеном

- **Геокодер городов**: сейчас `origin`/`destination` просто берутся как первые 3 буквы ввода
  в верхнем регистре — это заглушка. Нужен автокомплит через `/v1/city_iata`
  или упрощённый словарь популярных городов вылета.
- **Валидация дат**: нет проверки формата и того, что дата в будущем.
- **Rate limiting API**: Travelpayouts ограничивает частоту запросов — при росте базы подписок
  нужна очередь (например Celery + Redis) вместо простого `for` с `sleep(1)`.
- **Проверка справедливости данных**: `/v1/prices/cheap` — это кэш, а не live-поиск,
  подчеркните это в интерфейсе (уже сделано в текстах карточек), чтобы не вводить
  пользователей в заблуждение о точности цены на момент покупки.
- **Информация о визах**: для Туркменистана уместен отдельный блок `/visa_info` со ссылкой
  на официальные источники (посольство), так как большинству туристов нужна виза,
  а требования периодически меняются.
